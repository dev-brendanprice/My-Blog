Server and Client Technologies

Client Technologies

Front end technologies, or "client technologies", are tools and techniques that reside on the machine that are in control of the end-user. For example, right now you are using a computer to view this website, and as the end-user you are using the filsystem of the operating system to view this file locally instead of over internet.

In regards to utilizing these methods during web development, We can go down to the fundamentals which are defined as Hypertext Markup Languages, JavaScript and Cascading Style Sheets. These technologies, namely JavaScript as the outlining progrmaming language, are what a web developer will utilize to formalize a framework, build and improve upon to create the end product; a website.

Other front end technologies could include, but are not limited to; Frameworks that wrap the above described languages, Integrated Development Environments, Chrome DevTools (do Ctrl+Shift+i to view this websites' DevTools).

Relation to Presentation Layer of the OSI Model

All the above statements can be related to the presentation layer of the OSI model; Formalized to represent the telecommunications of a computer system with regards to its underlying features and functions. Said presentation layer is utlizied to parse the syntax that the developer is typing into the IDE and turning it into an output in the browser that the user can then view. This process works vice-versa.

Server Technologies

Back end technologies or "server technologoies", are tools and techniques that reside on the machine that is actively hosting the resource that you (the end-user) is requesting. If you are viewing this webpage, there are no active back end technologies because the files are static and locally configured, so there is no need for a server to host these files. For example, lets go to www.youtube.com, this website has countless back end technologies that are hosting the website files and other resources such as videos. These videos will only be sent back to the user based on a request for video resources.

During development of websites, developers may make use of push and pull technologies that push and pull versions from a remote location, so that multiple developers can work on the same project simultaneously. An example of this push and pull technology could be GitHub (which is what this website is based off). Back-end technologies come under this category as they are technologies that do no exist on the immediate local machine that the user is actively making use of.

In more simple terms, back-end technologies would also include web servers, which are machines that host website files that can be viewed by end-users via a browser and an internet connection. These web servers have other underlying technologies that it requires to operate such as domain name servers etc.

Relation to Application and Data Layer

The application layer characterizes protocols that are utilized when using server technologies. Protocols allow users to interface with machines that are connected to a network, or the internet, edit data, add new data or remove data. This is important as it allows web developer to make changes to files to a remote web server that is hosting the website that they are developing. These protocols are, but not limited to: File Transfer Protocol (FTP), Secure File Transfer Protocol (SFTP), Simple Mail Transfer Protocol (SMTP), HyperText Transfer Protocol (HTTP) and HyperText Transfer Protocol Secure (HTTPS).

The data layer characterizes a platform at which other processes are able to interface with and allows for an easy way to request and receive data. An example of this could be IP addresses and how computers will utilize this addresses to communicate with each other across a network or over the internet. Web developers would require their PC to have an IP address assigned to their machine to enable them to communicate with the web server where their website files reside.
